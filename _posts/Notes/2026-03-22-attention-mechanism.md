## 一句话版本

Attention 是一种让模型在处理某个元素时，**有选择地"看"序列中其他所有元素**，并根据"相关程度"加权提取信息的机制。

## 最直觉的类比

你在一个嘈杂的派对上，有 20 个人同时在说话。但你在跟朋友聊天，提到了"机器学习"。

这时你的耳朵会怎么做？
- 朋友说的话 → **高权重**（你在认真听）
- 旁边有人也在聊 AI → **中等权重**（相关话题，你会分一点注意力过去）
- 角落有人在聊今晚吃什么 → **低权重**（不相关，几乎忽略）

你的大脑在做的事：**根据当前关心的内容，动态分配注意力给不同的信息源**。

Attention 机制就是这个过程的数学版本。

## Attention 出现之前的问题

先讲它要解决什么问题。2014 年之前，机器翻译用的是 RNN encoder-decoder：

```
输入: "I love machine learning"
 ↓
 [Encoder RNN]
 ↓
 最终隐状态 h₄（一个固定长度的向量）
 ↓
 [Decoder RNN]
 ↓
输出: "我 喜欢 机器 学习"
```

问题在于 **h₄ 这一个向量要压缩整句话的所有信息**。就像让你用一张便签纸总结一本书，然后另一个人只根据这张便签翻译全书——信息必然丢失，句子越长越严重。

## Bahdanau Attention（2014）：让 Decoder "回头看"

Bahdanau 的核心idea：Decoder 在生成每个词的时候，不要只看那一个压缩向量，**让它回头看 Encoder 每个位置的隐状态，自己决定看哪里**。

```
Encoder 隐状态: h₁   h₂   h₃   h₄
                 ↑    ↑    ↑    ↑
                0.1  0.1  0.7  0.1  ← 注意力权重（由模型动态计算）
                 └────┴────┴────┘
                        ↓
                 加权求和 = context vector
                        ↓
                 Decoder 用这个来生成当前词
```

比如 Decoder 在生成"机器"这个词时：
- 给 "machine"（h₃）分配高权重 0.7
- 给其他词分配低权重
- 加权求和得到一个 context vector，主要包含 "machine" 的信息

下一步生成"学习"时：
- 权重重新计算，给 "learning"（h₄）最高权重

**关键点：每一步的权重都是动态计算的**，不是预先设定的。

## 用具体数字理解

假设 Encoder 有 4 个隐状态，每个是 3 维向量：

```
h₁ = [1.0, 0.5, 0.2]  ← "I"
h₂ = [0.3, 0.8, 0.1]  ← "love"
h₃ = [0.9, 0.2, 0.7]  ← "machine"
h₄ = [0.4, 0.6, 0.9]  ← "learning"
```

Decoder 当前要生成"机器"，它计算注意力权重：

```
α₁ = 0.05  （"I" 不太相关）
α₂ = 0.05  （"love" 不太相关）
α₃ = 0.80  （"machine" 很相关！）
α₄ = 0.10  （"learning" 有点相关）
—————————
总和 = 1.0
```

加权求和：

```
context = 0.05×h₁ + 0.05×h₂ + 0.80×h₃ + 0.10×h₄
        = 0.05×[1.0, 0.5, 0.2] + 0.05×[0.3, 0.8, 0.1]
          + 0.80×[0.9, 0.2, 0.7] + 0.10×[0.4, 0.6, 0.9]
        = [0.825, 0.285, 0.665]
```

这个 context vector 主要反映了 "machine" 的信息，Decoder 用它来生成"机器"。

## 那权重怎么算的？

这就是 attention 的核心。有两种主流方式：

### ① Additive Attention（Bahdanau, 2014）

用一个小神经网络来算相关度：

$$e_{t,i} = v^\top \tanh(W_1 h_i^{enc} + W_2 h_t^{dec})$$

$$\alpha_{t,i} = \frac{\exp(e_{t,i})}{\sum_j \exp(e_{t,j})}$$

直觉：把 encoder 隐状态和 decoder 隐状态都过一个网络，网络学习判断"这两个有多相关"。

### ② Dot-Product Attention（Transformer 用的）

直接用**点积**算相关度：

$$e_{t,i} = q_t \cdot k_i$$

$$\alpha_{t,i} = \text{softmax}(e_{t,i})$$

直觉：如果两个向量方向相似（点积大），说明它们相关。**更简单、更快、可以用矩阵乘法高效计算。**

## Attention 的通用公式

抽象成三个角色：

- **Query (Q)**：我在找什么？（当前正在处理的东西）
- **Key (K)**：每个候选有什么标签？（被查找的东西）
- **Value (V)**：每个候选的实际内容是什么？

```
步骤 1: 用 Q 和每个 K 计算相关度 → 得到权重
步骤 2: softmax 归一化 → 权重之和 = 1
步骤 3: 用权重对 V 加权求和 → 得到输出
```

**类比搜索引擎：**
- Q = 你的搜索词
- K = 每个网页的标题/关键词
- V = 每个网页的内容
- 搜索引擎用搜索词匹配标题（Q·K），按相关度排序，然后返回最相关的网页内容（V 的加权组合）

## Attention vs Self-Attention

这个区分很重要：

| | Attention（Cross-Attention） | Self-Attention |
|---|---|---|
| Q 来自 | 一个序列（如 Decoder） | 同一个序列 |
| K, V 来自 | 另一个序列（如 Encoder） | 同一个序列 |
| 在做什么 | "我对那边的序列哪里感兴趣？" | "我这个序列内部，谁和谁相关？" |

**Bahdanau 2014 的 attention** = Cross-Attention（Decoder 看 Encoder）

**Transformer 的核心创新** = Self-Attention（序列自己看自己），而且证明了**只用这个就够了**，不需要 RNN。

## 为什么 Attention 这么厉害

| 好处 | 解释 |
|------|------|
| **长距离直连** | 任意两个位置一步就能互相看到，不需要像 RNN 那样信息一步一步传 |
| **动态权重** | 不是固定规则，模型自己学习该看哪里 |
| **可解释** | 注意力权重直接告诉你模型在"看"哪里（论文最后的可视化） |
| **并行** | 所有位置的 attention 可以同时算（矩阵乘法） |

---

现在回到 Transformer 论文，它的核心主张就清晰了：

> 以前：RNN 做主力 + Attention 做辅助
>
> Transformer：**Attention 自己就是主力**，RNN 完全不需要

这就是"Attention Is All You Need"这个标题的含义。
