## 一句话版本

RNN 是一种**有"记忆"的神经网络**——它一步一步地读取序列，每读一个元素就更新自己的内部状态，用来记住"之前读过什么"。

## 直觉类比

想象你在**逐字读一句话**：

```
"我 今天 去 了 超市 买 了 苹果"
```

你的大脑在干什么？
- 读到"我" → 大脑记住：主语是"我"
- 读到"今天" → 更新记忆：时间是今天
- 读到"去了超市" → 更新记忆：我今天去了超市
- 读到"买了苹果" → 更新记忆：我今天去超市买了苹果

你的大脑在每一步都**用旧记忆 + 新输入 → 生成新记忆**。RNN 就是模拟这个过程。

## 数学公式

每个时间步 $t$：

$$h_t = f(W_h \cdot h_{t-1} + W_x \cdot x_t + b)$$

- $x_t$：当前时间步的输入（比如当前这个词的向量）
- $h_{t-1}$：上一步的隐状态（"记忆"）
- $h_t$：这一步的隐状态（更新后的"记忆"）
- $W_h, W_x, b$：可学习的参数
- $f$：激活函数（比如 tanh）

画成图：

```
x₁ x₂ x₃ x₄ x₅
 ↓  ↓  ↓  ↓  ↓
[h₀]→[h₁]→[h₂]→[h₃]→[h₄]→[h₅]
 ↑
 每一步都依赖前一步
```

## RNN 的两个致命问题

### 问题 1：无法并行

注意箭头 `→`——$h_2$ 必须等 $h_1$ 算完，$h_3$ 必须等 $h_2$ 算完... 这是**严格串行的**。

序列长度 1000 → 就要串行算 1000 步。GPU 有几千个核心，但只能用 1 个，其余全在等。

### 问题 2：长距离遗忘

假设序列很长：

```
"我 去年 在 北京 参加 了 一个 关于 人工智能 的 会议 ，会上 ...（省略50个词）... 那次 会议 的 主题 是 ___"
```

要填空，需要记住 50 多步之前的"人工智能"。但 $h_t$ 每一步都被覆写更新，早期信息会**逐渐被稀释**。

数学上，梯度在反向传播时要经过很多步的连乘，会出现：
- **梯度消失**：连乘很多 < 1 的数 → 梯度趋近 0 → 学不到远距离的依赖
- **梯度爆炸**：连乘很多 > 1 的数 → 梯度趋近 ∞ → 训练不稳定

## LSTM 和 GRU：RNN 的改良版

为了缓解长距离遗忘，研究者发明了两个变种：

### LSTM（Long Short-Term Memory，1997）

加了一个"细胞状态" $c_t$，像一条**传送带**，信息可以几乎不变地流过很多步：

```
c₁ ————→ c₂ ————→ c₃ ————→ c₄ （传送带，信息保留）
 ↕        ↕        ↕        ↕
h₁ ————→ h₂ ————→ h₃ ————→ h₄ （工作记忆）
```

通过三个"门"来控制：
- **遗忘门**：决定丢掉哪些旧信息
- **输入门**：决定写入哪些新信息
- **输出门**：决定输出哪些信息

### GRU（Gated Recurrent Unit，2014）

LSTM 的简化版，把三个门合并成两个（重置门 + 更新门），参数更少，效果差不多。

## 但 LSTM/GRU 依然没解决并行问题

不管怎么加门控，**串行计算的本质没变**——$h_t$ 还是要等 $h_{t-1}$。

这就是 Transformer 要解决的核心问题：

| | RNN/LSTM | Transformer |
|---|---------|-------------|
| 并行性 | ❌ 串行 | ✅ 完全并行 |
| 长距离依赖 | 困难（梯度消失） | O(1) 直接连接 |
| 计算效率 | 序列越长越慢 | 可以充分利用 GPU |

## 一个有趣的历史细节

Transformer 论文的 Introduction 提到：

> *"attention mechanisms are used in conjunction with a recurrent network"*

2017 年之前，大家觉得 attention 只是 RNN 的辅助工具。Transformer 说：**不需要 RNN，attention 自己就够了**。这就像有人说"汽车不需要马，发动机自己就够了"——事后看很合理，但当时很激进。
